Who this is for

Product managers who are technically comfortable — you understand how software works conceptually, you're not afraid of a terminal — but you don't write code professionally. By the end of this guide, you'll have a fully working AI-assisted development environment on Windows and a clear understanding of how to use it to prototype and build.

What you'll have when you're done
  • ✓ A local setup where you can describe what you want to build in plain English, have AI generate the code, and run it instantly.
  • ✓ Git, VS Code, and Claude Code installed and configured — no Node.js required.
  • ✓ A GitHub account connected and a Vercel deployment pipeline set up.
  • ✓ A repeatable daily workflow for prototyping, testing, and publishing to a live URL you can share with anyone.

Setup guide
1

Install Git for Windows

Download the latest x64 version from git-scm.com/downloads/win and run the installer with all default settings. The "Add Git to PATH" option is checked by default — leave it checked. Install this before Claude Code.

After installing, verify with:

PowerShell 
git --version    # e.g. git version 2.53.0.windows.2

Why this comes first: Claude Code uses Git Bash internally to execute commands on Windows. Without it, Claude Code installs but can't function.

Working with PowerShell

Press the Windows key, type PowerShell, and hit Enter to open it. All commands in this guide get pasted directly into that window. If something doesn't work or you get an error you don't recognise, paste the error into Claude.ai and ask — it knows exactly what's happening and will give you the exact command to fix it.

2

Install Claude Code

Open PowerShell (search it in the Start menu — no need to run as Administrator) and paste this command. No Node.js required.

PowerShell 
irm https://claude.ai/install.ps1 | iex

⚠ If claude isn't recognised after install, close and reopen PowerShell. If it still doesn't work, ask Claude for the PATH fix command — it takes 30 seconds.

Subscription required: Claude Code needs a Claude Pro account ($20/mo) or higher. The free tier does not include access.

3

Install VS Code

Download from code.visualstudio.com and click the Windows button — it auto-detects your OS. Run the installer with default settings. This gives you a visual file explorer and a built-in terminal so you can see and build in one window.

PowerShell 
code --version   # confirms VS Code is in your PATH
4

Create a GitHub account

Go to github.com and sign up for a free account. GitHub is cloud storage for your code — it backs it up, makes it shareable with collaborators, and connects to your deployment pipeline. Choose your username thoughtfully; it appears in your repository URLs.

If you're setting up a fresh account, now is the easiest time to choose the username you want — before any repositories exist.

5

Create your first project folder and launch Claude Code

Open VS Code, press Ctrl + ` to open the built-in terminal, and run these commands. Claude Code will authenticate via browser on first launch.

PowerShell 
# Create a dedicated workspace
mkdir C:\Users\[yourname]\Projects
cd    C:\Users\[yourname]\Projects

# Launch Claude Code
claude

On first launch: select a display theme, then authenticate via the browser window that opens. You'll see a welcome screen with your account name and a > prompt. You're ready.

⚠ Before your first save, tell Git who you are.

This takes 30 seconds and only needs to be done once. Skip it and your first publish attempt will silently fail — no error you can easily diagnose, just nothing making it to GitHub or Vercel.

Run these two lines in PowerShell, replacing the placeholders with your actual GitHub email and username:

PowerShell 
git config --global user.email "you@example.com"
git config --global user.name  "yourGitHubUsername"

Every change you save to GitHub will now be stamped with your name. Collaborators and developers looking at the project history will see that you built it.

6

Set up Vercel for publishing

Go to vercel.com and sign up with your GitHub account. Once connected, Vercel watches your repositories and automatically publishes updates within about 30 seconds of saving code to GitHub. No configuration needed for standard web projects.

Every project you push to GitHub gets a URL like your-project.vercel.app automatically. Share it with stakeholders, users, or your dev team for feedback on a real working build.
